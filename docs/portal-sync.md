# Портал SwimBuoy: синк маршрута и отправка трека с часов

Портал — **swimbuy.iron-siber.ru** (бэкенд + сайт в каталоге `backend/`).
Watch-app (Garmin Connect IQ) получает маршрут с портала и автоматически
отправляет туда трек заплыва. Аналог загрузки маршрутов в Coros, но через
собственный портал, а не телефонное приложение.

## Что добавлено в watch-app

| Файл | Изменение |
|------|-----------|
| `connect-iq/manifest.xml` | разрешение `Communications` |
| `connect-iq/resources/properties.xml` | свойства `backendUrl`, `athleteToken`, `syncRouteId`, `autoUpload`, `trackSampleSec` |
| `connect-iq/resources/settings/settings.xml` | настройки в Garmin Connect (ввод токена/URL/маршрута) |
| `connect-iq/source/BackendClient.mc` | сетевой клиент: синк маршрута + отправка трека |
| `connect-iq/source/RouteEngine.mc` | загрузка маршрута из `Storage` (скачан), фолбэк на вшитый JSON |
| `connect-iq/source/SwimBuoyView.mc` | буфер GPS-точек, отправка трека на финише, синк при запуске |

## Настройка на часах (Garmin Connect → SwimBuoy → Settings)

1. **Portal URL** — `https://swimbuy.iron-siber.ru` (по умолчанию).
2. **Athlete token** — 8-символьный токен спортсмена (выдаёт админка портала;
   буквы/цифры, вводится как пароль).
3. **Route ID to sync** — ID маршрута с портала (со страницы маршрута на сайте).
4. **Auto-upload track** — отправлять трек автоматически после заплыва (по умолч. вкл).
5. **Track sample, sec** — шаг буферизации GPS-точек (по умолч. 3 с).

## Как это работает

**Загрузка маршрута.** При запуске приложения, если задан `Route ID` и токен,
часы делают `GET /api/watch/routes/{id}`, кэшируют ответ в `Storage("activeRoute")`
и `RouteEngine` перечитывает маршрут. Без сети используется последний кэш или
вшитый в сборку `lake_demo.buoy_route.json`.

**Отправка трека.** Во время заплыва GPS-точки буферизуются (`{t,lat,lon}`,
шаг = `Track sample`, максимум 2000 точек). По завершении (кнопка «назад» →
сохранить активность) часы шлют `POST /api/watch/activities` с маршрутом и
буфером. Полезная нагрузка пишется в `Storage("pendingUpload")` и
переотправляется при следующем запуске, если ответ не пришёл (например, не было
связи с телефоном). Портал строит отчёт по коридору и показывает его на сайте.

**Ручная отправка.** Кнопка **Menu** на экране заплыва открывает меню с пунктом
**«Отправить трек»**. Подпись пункта показывает состояние связи («связь есть» /
«нет связи»); отправка срабатывает только при подключённом телефоне, иначе на
экране появляется «Нет связи с телефоном». Там же доступен «Пропустить буй».

> Для сетевых запросов часам нужен телефон с интернетом по Bluetooth (как и у Coros).
> Параллельно часы продолжают писать обычную FIT-активность — её можно выгрузить
> и вручную загрузить на портал (GPX/TCX/FIT), если автоотправка недоступна.

## Где взять токен и Route ID

- **Токен спортсмена** — веб-админка `…/#/admin` (логин `admin` / `sw1mBu7`) →
  вкладка «Спортсмены» → создать → скопировать токен. Или вывод `backend/scripts/seed.py`.
- **Route ID** — открыть маршрут на сайте; ID в адресе (`#/routes/<id>`) и в
  ответе `GET /api/watch/routes`.
