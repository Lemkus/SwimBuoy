# Product decisions (MVP)

## 1) Навигация

- Режим по умолчанию: `point_proximity`.
- Не используем `off-course` от линии как основной сигнал.
- Курс GPX/FIT — опционально для превью/совместимости, не ядро логики.

## 2) UX на часах

- Детали: [`docs/spec-watch-ui.md`](spec-watch-ui.md).
- Крупные **метры** до активной точки.
- **Полоса коридора** (позиция по XTE) на дистанции; **стрелка** только вблизи буя (≤ 50 m).
- Во время заплыва показываем только текущую цель.

## 3) Точки и порядок

- `points` (координаты) стабильны.
- `session.order` динамический.
- Режимы порядка для MVP: `fixed`, `random`, `shuffle_each_lap`.
- `nearest_next` и более сложные режимы — после базовой стабилизации.

## 4) Засчитывание точки

- Условие: `distance < arrivalRadiusM` и удержание `dwellSec`.
- Базовые дефолты: `arrivalRadiusM = 20`, `dwellSec = 4`.

## 5) Технический приоритет

1. Стабильные метры и полоса коридора на FR955 (см. `docs/spec-watch-ui.md`).
2. Простая смена точки.
3. Полевые тесты на воде.
4. Только потом расширения (миссии, ghost, вибрация).

## 6) Тестирование на реальном треке

- Источник GPS: FIT с FR955 (Garmin Connect → «Экспортировать файл»).
- Для тестового заплыва: Activity Recording = «каждую секунду» (не «интеллектуально»).
- Симулятор: FIT/GPX Playable File в Activity Data; без ▶ Play GPS в app не идёт.
- Буи ставим по точкам реального трека; координаты — в `connect-iq/resources/jsonData/*.json`.
- Координаты озера: полевой круг P1–P5 из TCX 24.05.2026 (см. `routes/field_walk_2026-05-24.md`, `docs/SESSION_SUMMARY.md`).
