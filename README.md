# SwimBuoy

MVP-проект для тренировки плавания на открытой воде с виртуальными точками.

## Цель MVP

Сделать приложение для Garmin FR955 (Connect IQ), где пловец видит:
- крупное число метров до текущей точки;
- стрелку направления на текущую точку.

Точки на озере фиксированы по координатам, а порядок прохождения может меняться.

## Основная логика

- Навигация к точке (`point_proximity`), а не по отклонению от линии курса.
- Засчитывание точки: вход в радиус + удержание в радиусе несколько секунд.
- В заплыве показывается только активная цель; следующая включается после взятия текущей.

## Структура (первый этап)

- `connect-iq/` — код приложения на Monkey C.
- `routes/` — JSON-маршруты с точками и порядком.
- `backend/` — портал **swimbuy.iron-siber.ru** (FastAPI + сайт): маршруты,
  загрузка/приём треков, отчёты по коридору, веб-админка. См. `backend/README.md`.
- `docs/decisions.md` — ключевые решения, чтобы не терять контекст.
- `docs/portal-sync.md` — как часы получают маршрут и шлют трек на портал.

## Портал и часы

- Создавайте/редактируйте маршруты на сайте; часы тянут выбранный маршрут по сети
  (`Route ID` в настройках приложения) и автоматически отправляют трек заплыва.
- Отчёт по тренировке (трек + коридор + взятые буи) строится на портале и
  доступен по публичной share-ссылке. Треки можно также загружать вручную
  (GPX/TCX/FIT). Подробнее: `backend/README.md`, `docs/portal-sync.md`.

## Что не в MVP

- Физические буи.
- Плавательные очки.
- Сложная вибро-логика как обязательная часть UX.
- Полноценная публикация в магазине на первом шаге.

## Сборка (Connect IQ)

Требуется [Connect IQ SDK](https://developer.garmin.com/connect-iq/sdk/) и VS Code с расширением **Monkey C**.

```powershell
connect-iq\setup-sdk-path.bat   # после перезагрузки Windows
# Ctrl+Shift+B или F5 — один .prg: connect-iq\bin\SwimBuoy.prg
# (в корне bin\ — та же папка, junction; см. scripts\setup-build-link.bat)
# Вручную:
X:\bin\monkeyc.bat -f connect-iq\monkey.jungle -o connect-iq\bin\SwimBuoy.prg -y connect-iq\developer_key.der -d fr955 -w
```

Симулятор: **Settings → Set Position**, затем **Simulation → Activity Data → ▶ Play** (или Load FIT).

### Маршрут

Демо-маршрут лежит в `connect-iq/resources/jsonData/lake_demo.buoy_route.json`.
Чтобы сменить точки — отредактируй JSON и пересобери приложение.
Эталонный формат — `routes/example.buoy_route.json`.

### Экран часов

- Крупные **метры** до активной точки.
- **Стрелка** на точку (относительно GPS-курса; без курса — абсолютный азимут).
- Зелёная стрелка = внутри радиуса, идёт отсчёт `dwell`.
- После `arrivalRadiusM` + `dwellSec` — переход к следующей точке из `session.order`.
