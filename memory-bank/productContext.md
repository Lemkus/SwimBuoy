# Product Context — SwimBuoy

Обновлено: 2026-06-08

## Пользовательский сценарий

1. Один раз смотришь карту точек на озере — запоминаешь **где** буи.
2. В заплыве плывёшь по памяти, редко смотришь часы.
3. На часах — метры и стрелка на **активный** буй.
4. Следующая точка появляется после взятия текущей (радиус + dwell).

## Модель данных

| Сущность | Поведение |
|----------|-----------|
| `points` | Фиксированные координаты (P1, P2, …) |
| `session.order` | Порядок обхода — по режиму |
| Режимы порядка | `fixed`, `random`, `shuffle_each_lap` |

## Текущий маршрут

**Аунеланлахти (Токсово)** — приложение `SB_Toksovo`, JSON в сборке: `connect-iq/resources/jsonData/lake_demo.buoy_route.json`.
